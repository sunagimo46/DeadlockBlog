# Deadlock 攻略ブログ — 実装方針

## Context

Deadlock（Valve）の攻略ブログを構築する。YouTube、Reddit、Wikiから話題の情報を定期収集し、GitHub Issue として提示。ユーザーが選んだトピックについて Claude Code（claude-code-action）が記事を自動生成し、PRを作成する。Maxプランを活用し追加コスト $0 で運用する。

---

## 1. 全体アーキテクチャ

```
GitHub Actions Cron（毎日）
  │
  ▼
Python スクリプト（データ収集）
  │  YouTube RSS / Reddit JSON API / MediaWiki API
  │  ※ すべて APIキー不要
  ▼
GitHub Issue 自動作成
  │  「本日の話題トピック」一覧
  │
  ▼  ユーザーが確認・選択
Issue コメント「@claude ○○の記事を書いて」
  │
  ▼
claude-code-action 起動（Max プラン OAuthトークン）
  │  CLAUDE.md に従い記事生成
  │  Web検索で詳細情報を補完
  ▼
PR 自動作成（src/content/blog/ に .md）
  │
  ▼  ユーザーがレビュー & マージ
Netlify 自動ビルド・デプロイ
```

---

## 2. デザイン方針

- **テーマ**: ダーク基調（Deadlock のゲーム雰囲気に合わせる）
- **カラーパレット**: ダークグレー背景 + アクセントカラー（ゲーム内UIに寄せた配色）
- **フォント**: ゴシック系（Noto Sans JP 等）、見出しはやや太め
- **レイアウト**: カード型の記事一覧、サイドバーにカテゴリ・タグ
- **Tailwind CSS v4**: `@theme` でカスタムカラートークンを定義

## 6. GitHub Actions ワークフロー

### 5-1. collect-topics.yml（定期データ収集）

```yaml
name: 話題トピック収集

on:
  schedule:
    - cron: '0 0 * * *'        # 毎日 UTC 0:00（JST 9:00）
  workflow_dispatch:             # 手動実行可能

jobs:
  collect:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install -r scripts/requirements.txt
      - run: python scripts/collect.py
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

**collect.py の処理:**
1. YouTube RSS、Reddit JSON、Wiki API から情報取得
2. 各ソースの話題トピックを整形
3. `gh issue create` で Issue を作成（ラベル: `topic-collection`）

### 5-2. claude.yml（記事生成）

```yaml
name: Claude 記事生成

on:
  issue_comment:
    types: [created]
  issues:
    types: [opened, labeled]

jobs:
  claude:
    if: contains(github.event.comment.body, '@claude')
    runs-on: ubuntu-latest
    steps:
      - uses: anthropics/claude-code-action@v1
        with:
          claude_code_oauth_token: ${{ secrets.CLAUDE_CODE_OAUTH_TOKEN }}
          trigger_phrase: "@claude"
```

## 7. CLAUDE.md（記事生成ガイドライン）

Claude Code が記事を生成する際に従うルールを定義する。

### 主な内容

- **言語**: 日本語
- **ファイル命名**: `src/content/blog/YYYY-MM-DD-slug.md`
- **フロントマター**: title, description, pubDate, tags, hero（キャラ名）, category
- **記事カテゴリ**: ヒーロー攻略、パッチノート解説、メタ考察、アイテムガイド等
- **記事構成テンプレート**: カテゴリごとに定義
- **参考情報源**: Deadlock Wiki、Reddit、YouTube の URL
- **執筆スタイル**: 初心者にも分かりやすく、実戦的な内容
- **品質チェック**: 日本語であること、フロントマターの必須項目、見出し構成

## 8. Content Collections スキーマ

```typescript
// src/content/config.ts
import { defineCollection, z } from 'astro:content'

const blog = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    description: z.string(),
    pubDate: z.coerce.date(),
    updatedDate: z.coerce.date().optional(),
    tags: z.array(z.string()),
    category: z.enum([
      'hero-guide',      // ヒーロー攻略
      'patch-notes',     // パッチノート解説
      'meta',            // メタ考察
      'item-guide',      // アイテムガイド
      'tips',            // Tips & テクニック
    ]),
    hero: z.string().optional(),     // 関連ヒーロー名
    draft: z.boolean().default(false),
  }),
})

export const collections = { blog }
```

- `collect-topics.yml`（定期収集）作成
- `claude.yml`（記事生成）作成

- Netlify 連携設定

## 10. 運用フロー（日常の使い方）

```
毎日（自動）
  → 「本日の話題トピック」Issue が作成される

ユーザー（随時）
  → Issue を確認
  → 書いてほしいトピックに @claude でコメント
    例: 「@claude このパッチノートの解説記事を書いて」

Claude Code（自動）
  → 記事を生成して PR を作成

ユーザー（随時）
  → PR をレビュー、必要なら修正依頼
  → マージ → Netlify が自動デプロイ
```

---

## 11. 検証方法

各フェーズ完了時に以下を確認:

- **Phase 1**: `npm run dev` でブログが表示される。サンプル記事が一覧・詳細で閲覧可能
- **Phase 2**: `python scripts/collect.py` でデータ取得 → コンソールにトピック一覧が表示される
- **Phase 3**: GitHub 上で手動 workflow_dispatch → Issue が作成される。Issue に @claude コメント → PR が作成される
- **Phase 4**: PR マージ後、Netlify で本番サイトが更新される
